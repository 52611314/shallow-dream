## 1.4：RAG知识库构建（2026.5.25）

看了会操作系统看不下去了，王道讲的还行但是这玩意本身巨绕，还是做项目算了。

不过最近项目也越来越难了，很难看懂啊，而且也近乎于完全依赖ai了，有空多看看吧。

* **创建文档表与向量扩展** ：定义 `Document` 模型，使用 `pgvector` 添加 384 维向量列，支持语义检索。
* **集成 Embedding 模型** ：接入 `sentence-transformers/all-MiniLM-L6-v2` 模型，将文档片段和用户问题转换为向量。
* **实现文档导入脚本** ：编写 `ingest.py`，支持 `.txt` / `.md` 文件的分块、向量生成与批量入库。
* **优化文档分块策略** ：从按字符切分升级为按段落和句子边界切分，保持语义完整性，提升检索质量。
* **异步向量生成** ：将同步的 `model.encode` 用 `asyncio.to_thread` 包装，避免阻塞 FastAPI 事件循环。
* **封装 RAG 检索函数** ：在 `database.py` 中实现 `search_documents`，通过向量余弦距离检索最相关的文档片段。
* **增强对话接口** ：修改 `/chat` 端点，在调用大模型前先执行向量检索，将相关文档拼接到上下文提示词中。
* **提供文档管理 API** ：新增 `documents.py` 路由，包含 `/upload`（上传导入）、`/list`（列出文档）、`/clear`（清空知识库）接口。
* **完成集成测试** ：使用 Apifox 上传小组项目依赖文件，验证 RAG 能准确识别文档内容并基于上下文回答问题。
